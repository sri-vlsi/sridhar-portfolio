Developed an advanced security framework for PCIe systems that prevents unauthorized memory access in multi-CPU environments. The solution implements a token-based access control mechanism using PSS 2.0 features to secure configuration space and memory regions.

Technical Details

  • Implemented resource locking and sharing mechanisms for multi-CPU synchronization
  • Developed byte-addressable region management with parameterized Traits

  • Designed secure configuration space allocation algorithms
  • Mitigated race conditions in PCIe enumeration process
  • Utilized Siemens QPS and Synopsys VCPS for verification
PSS 2.0 System Verilog Linux Red Hat 8.1 Siemens QPS Synopsys VCPS DSL

Key Achievements

Successfully enhanced protection against unauthorized I/O operations while maintaining system performance. Reduced configuration space access conflicts by 92% in multi-CPU test scenarios. This work formed the basis for my conference paper at DVCon Taiwan 2025 titled "Secure Multi-CPU Memory Access in PCIe via Tokenized Address Space Management".
