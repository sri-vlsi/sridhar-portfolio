Data Integrity and Data Security for PCIe using PSS Language

Company: Silicon Interfaces, Mumbai

Duration: Jan 2025 - Current

Team Size: 2 Engineers

Technologies: PSS 2.0, Linux Red Hat 8.1, Siemens QPS, Synopsys VCPS

Project Description

Developed a token-based framework to secure PCIe memory access in multi-CPU systems using byte-addressable regions and synchronized IPC-based access control. Ensured secure CF8/CFC register access and mitigated race conditions, enhancing protection against unauthorized I/O operations.

Technical Details

  • Implemented a robust solution for synchronized access to CF8/CFC ports in multi-CPU systems
  • Utilized PSS 2.0 features like resource locking, sharing, and constraints
  • Leveraged address space management and byte addressability
  • Implemented parameterized Traits to prevent race conditions
  • Enabled secure, efficient configuration space allocation

Methodologies

The project employed Portable Stimulus Standard (PSS) for verification, with the following key aspects:

  • DSL-based verification environment
  • Token-based access control mechanism
  • Multi-CPU synchronization protocols
  • Configuration space security enhancements

Results

The implemented solution successfully:

  • Prevented unauthorized access to PCIe configuration space
  • Eliminated race conditions in multi-CPU environments
  • Improved system security while maintaining performance
  • Served as the basis for a conference paper presented at DVCon Taiwan 2025

PCIe Gen 3.0 Design, Implementation, Verification

Company: Silicon Interfaces, Mumbai

Duration: Apr 2024 - Dec 2024

Team Size: 2 Engineers

Technologies: System Verilog, UVM, Linux Red Hat 8.1, QuestaSim, VCS

Project Description

PCI Express is a high-speed serial bus standard designed to replace older PCI/PCI-X interfaces. PCIe 3.0 uses 128b/130b encoding, replacing the 8b/10b encoding of PCIe 2.0, which reduces overhead to ~1.5% and achieves a maximum bandwidth of up to 985 MB/s per lane per direction.

My Contributions

  • Designed and verified PCIe enumeration logic using System Verilog
  • Implemented linked list-based structure to dynamically manage and represent the PCIe device hierarchy
  • Simulated the complete enumeration flow including:
    • Bus/Device/Function (BDF) number assignment
    • Configuration space traversal
    • Multifunction device detection
  • Performed verification using System Verilog constraints to test diverse topology scenarios
  • Collaborated in debugging complex enumeration issues
  • Optimized device insertion algorithms

Technical Highlights

  • Developed UVM-based verification environment
  • Implemented comprehensive testbench with scoreboard and coverage collection
  • Created constrained random tests for various PCIe topologies
  • Developed functional coverage models for enumeration process
  • Analyzed and optimized performance of enumeration algorithm
